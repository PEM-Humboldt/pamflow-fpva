# Birds Index Pipeline

This pipeline computes a normalized bird community index for each deployment based on validated species detections.

## Purpose

The `birds_index` pipeline produces a summary index of detected bird species richness per deployment, using only species observations that meet validation thresholds.

## Input datasets

- `thresholds@pandas`: species-specific validation thresholds generated by the species detection pipeline, using experts validations on machine-made detections. 
- `observations@pamDP`: detection observations using BirdNet 2.4.

## Output dataset

- `birds_index@pandas`: a CSV dataset with one row per `deploymentID` and a normalized `indice` value.

## What it does

1. Restricts observations to those species present in the validated thresholds.
2. Merges species-specific thresholds into the observations.
3. Marks detections where `classificationProbability` is larger than the species-specific threshold.
5. Counts unique species per deployment.
6. Normalizes the count by the total number of validated species.

## Where results are stored

The output dataset is configured in `conf/base/catalog/birds_index.yml` and writes to:

`data/output/birds_index/avifauna_interior_bosque.csv`

## Pipeline usage

This pipeline is registered as `birds_index` in `src/pamflow/pipeline_registry.py` and can be executed as part of the Kedro workflow.

Example:

```bash
kedro run --pipeline birds_index
```
